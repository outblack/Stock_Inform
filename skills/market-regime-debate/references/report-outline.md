## Style Rules

- use report-style headings, not role names
- keep the main body reader-facing and synthesis-heavy
- do not dump raw role notes into the report unless the user explicitly asks for an appendix
- update the original stub file in place instead of creating a second markdown report
- if any extra markdown draft was created accidentally, delete it before finishing
- for any non-trivial run, default to a substantial long-form report rather than a short memo
- when the report language is Korean, use plain and reader-friendly Korean titles and headings
- for Korean output, avoid awkward labels like `시장 스냅샷`, `글로벌 매크로 뷰`, and `한국 매크로 뷰` when `지금 먼저 봐야 할 숫자`, `세계 경제 흐름`, and `한국 경제 흐름` would read better
- when the evidence base is rich, expand each major section into multiple real paragraphs instead of one short paragraph and a few bullets
- a non-trivial report should usually read like a deep briefing note, not a compact blog post
