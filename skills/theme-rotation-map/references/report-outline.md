# Report Outline

Use this order unless the user asks for another format:

1. `Executive Verdict`
2. `Request Snapshot`
3. `Today Reference Frame`
4. `Market Pulse`
5. `Theme Ranking Table`
6. `Why The Top Themes Rank Here`
7. `Top Theme Deep Dives`
8. `Integrated Leader Stock Playbook`
9. `Rotation And Relay Scenarios`
10. `Crowded, Fading, And Rejected Themes`
11. `Debate Synthesis And Evidence Check`
12. `Action Matrix`
13. `Key Risks And Invalidation Triggers`
14. `Source Log`

## Single-Report Contract

- one invocation should produce one final report
- leader-stock analysis belongs inside the same final report
- do not create a separate leader-stock sidecar report unless the user explicitly requests it
- do not dump raw role-by-role notes into the main report body
- update the original stub file in place instead of creating a second markdown report
- if any extra markdown draft was created accidentally, delete it before finishing

## Leader Stock Playbook Requirements

For every leader the report elevates to `tradable` or `primary`, include:

- exact price snapshot date and time when available
- why it qualifies as the true leader
- trend and technical posture
- support, resistance, breakout, and invalidation levels
- buy-now, buy-on-pullback, wait, trim, and exit conditions when the evidence supports them

## Style Rules

- use report-style headings, not role names
- keep the main body reader-facing and synthesis-heavy
- put any raw debate appendix behind an explicit user request
- for any non-trivial run, default to a substantial long-form report rather than a short memo
- when the report language is Korean, translate the title and section headings into natural Korean
